Module 5 — Behaviour

JavaScript
The Language of the Web

JavaScript is what turns a static document into a living application. It responds to user actions, updates content without page reloads, validates forms, fetches data from APIs, and — with Node.js — runs on the server too. Mastering JavaScript is the single biggest step toward becoming a capable full stack developer.

Overview

What I Learned About JavaScript

JavaScript was where this module felt most practical to me. I could immediately see the effect of what I was writing — a button click that opened a menu, a scroll event that triggered an animation, a setTimeout that built a typing effect character by character. That immediate feedback loop made it easier to learn by experimenting.

The shift from basic JavaScript to good JavaScript was harder. I learned to prefer const and let over var, to use addEventListener over inline handlers, and to favour IntersectionObserver over scroll events with no debouncing. These distinctions matter for performance and maintainability at scale.

Fundamentals

Core JavaScript Concepts

Variables

Use const by default for values that won't be reassigned, let when they will. Avoid var — its function scope creates subtle bugs that let and const's block scope prevent.

Functions

Reusable logic blocks. Arrow functions (() => {}) are shorter and don't create their own this, making them preferable for callbacks and event handlers.

DOM Manipulation

The Document Object Model is a live tree of every element on the page. JavaScript can read, modify, add, or remove nodes — enabling dynamic content updates without reloading.

Events

addEventListener attaches behaviour to user actions (click, scroll, input, keydown). Event delegation — one listener on a parent — is more efficient than individual listeners on every child.

Promises & Async/Await

Asynchronous operations (fetching data, reading files) return Promises. async/await makes this code read linearly, which is far easier to reason about than nested callbacks.

Modules (ES6)

import and export split JavaScript into separate, scoped files. This avoids polluting the global scope and makes large codebases maintainable.

Code in Practice

From This Portfolio

Rather than a toy example, here is the actual scroll-reveal logic used on this page. It uses IntersectionObserver — the modern, performant alternative to listening on the scroll event. 

// Select all elements that should animate on scroll
const cards = document.querySelectorAll('.feature-card, .advantage-item');

// IntersectionObserver fires when elements enter or leave the viewport
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible'); // triggers CSS transition
      observer.unobserve(entry.target);      // stop watching once done
    }
  });
}, {
  threshold: 0.12,         // 12% of the element must be visible
  rootMargin: '0px 0px -40px 0px'  // trigger 40px before the bottom
});

// Add staggered transition delays for visual interest
cards.forEach((el, i) => {
  el.style.transitionDelay = (i % 4) * 0.08 + 's';
  observer.observe(el);
});

// Why not scroll events?
// window.addEventListener('scroll', handler) fires dozens of times per second.
// IntersectionObserver is browser-optimised and fires only when needed.

    Applications

    What JavaScript Enables

    ✅
    Form Validation

    Check input client-side before a network request. Immediate feedback is faster and more user-friendly than a server round-trip for simple errors.

    🌐
    API Data Fetching

    The fetch() API retrieves JSON from external services — weather, maps, news — and renders it into the page without a reload.

    🎭
    UI Interactions

    Dropdowns, modals, tab panels, sliders, accordions — any interactive component that requires hiding or showing content dynamically.

    📊
    Data Visualisation

    Libraries like Chart.js and D3.js use the Canvas and SVG APIs to render interactive charts, graphs, and maps from raw data.

    🖥️
    Server-Side (Node.js)

    Node.js runs JavaScript outside the browser, enabling the same language to handle HTTP requests, database queries, and file operations on the server.

    📱
    Mobile Apps

    React Native and frameworks built on it allow JavaScript developers to build iOS and Android apps with a largely shared codebase.
