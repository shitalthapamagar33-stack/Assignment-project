Module 3 — Presentation

CSS
Cascading Style Sheets

CSS is what transforms raw HTML into a designed interface. It controls every visual decision — colour, spacing, typography, layout, motion — and with modern techniques like custom properties and Grid, it has become a genuinely powerful design tool in its own right.

Overview

What I Learned About CSS

My early CSS was mostly trial and error — changing colours until something looked right. This module pushed me to think more systematically. I began using CSS custom properties (variables) to define a consistent design token system, which meant a single colour or spacing value could be updated in one place and reflect everywhere.

The most significant shift was understanding layout. For years, developers fought browser inconsistencies with floats and tables. Flexbox solved one-dimensional layout cleanly, and CSS Grid solved two-dimensional layout. Learning both — and knowing which to reach for — changed how I think about page structure entirely.

Delivery Methods

Types of CSS

Inline CSS

Applied directly on an element using the style attribute. Useful for rapid testing or one-off overrides, but poor for maintainability — it mixes concerns and cannot be reused.

Internal CSS

Written in a <style> block in the document head. Appropriate for single-page prototypes, but not scalable across a multi-page site.

External CSS

Stored in a separate .css file linked via <link>. The professional standard: cacheable, reusable across pages, and keeps HTML clean.

Practical Examples

Key CSS Techniques

The examples below show patterns I used when building this very portfolio. 

/* Custom properties create a reusable design token system */
:root {
  --clr-gold:    #c9a84c;
  --clr-bg:      #0a0f1e;
  --clr-text:    #e2e8f0;
  --sp-lg:       2.5rem;
  --r-md:        12px;
}

/* Flexbox: one-dimensional, row or column */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* Grid: two-dimensional, rows AND columns */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: var(--sp-lg);
}

/* Responsive: adapt layout at breakpoints */
@media (max-width: 768px) {
  .card-grid {
    grid-template-columns: 1fr;
  }
}

        Modern CSS
        
Techniques Worth Knowing

      

      

        

          🎨
          
Custom Properties
Variables that cascade through your stylesheet, making global design changes a single-line edit instead of a find-and-replace.

        

        

          📐
          
Flexbox
Ideal for navbars, button groups, and any layout where items need to align along a single axis with even spacing.

        

        

          🔲
          
CSS Grid
Two-dimensional layout for card grids, page scaffolding, and complex arrangements that Flexbox can't express cleanly.

        

        

          📱
          
Media Queries
Rules that apply only at certain viewport widths, allowing a layout to reflow from a multi-column desktop view to a single-column mobile view.

        

        

          ✨
          
Transitions & Animations
Smooth visual changes on hover or class addition. Best used sparingly — animation should communicate, not just decorate.

        

        

          🔭
          
Clamp() & Fluid Type
Scale font sizes smoothly between a minimum and maximum value based on viewport width, eliminating brittle breakpoint-based font changes.
