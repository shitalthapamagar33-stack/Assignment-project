Module 4 — Frameworks

Bootstrap
The CSS Framework

Bootstrap removes the repetitive groundwork of building responsive layouts from scratch. Rather than replacing CSS knowledge, it accelerates teams that already understand CSS — you still need to know what you're doing; Bootstrap just lets you do more of it faster.

Overview

What I Learned About Bootstrap

Bootstrap was originally developed at Twitter to bring consistency to their internal tools. It became open-source in 2011 and has since become one of the most widely used CSS frameworks on the internet. Understanding Bootstrap well — including when not to use it — is a practical skill for working in almost any web development team.

The clearest lesson I took from this module: Bootstrap is a starting point, not a finish line. Projects that rely on it exclusively tend to look identical. The skill is knowing which Bootstrap defaults to keep, which to override, and which to replace with custom CSS entirely.

Core Concept

The 12-Column Grid

Bootstrap divides every row into 12 equal columns. By assigning column classes, you control how many of those 12 a given element occupies — and how that changes at different breakpoints. This was the concept that made responsive layout immediately intuitive for me. 

<!-- Bootstrap grid: 3 equal columns on medium+ screens, stacked on mobile -->
<div class="container">
  <div class="row g-3">
    <div class="col-12 col-md-4">
      <div class="card p-3">Column 1</div>
    </div>
    <div class="col-12 col-md-4">
      <div class="card p-3">Column 2</div>
    </div>
    <div class="col-12 col-md-4">
      <div class="card p-3">Column 3</div>
    </div>
  </div>
</div>

<!-- Result: 3 cards side-by-side on desktop, stacked on mobile -->

        Ready-Made UI
        
Key Bootstrap Components

      

      

        

          
Buttons

          
Classes like .btn .btn-primary provide styled, accessible buttons with hover and focus states built in — including keyboard focus visibility.

        

        

          
Cards

          
Flexible content containers with optional headers, footers, and image areas. Useful for feature grids, product listings, and portfolio layouts.

        

        

          
Navbar

          
A responsive navigation bar that collapses into a hamburger menu on small screens — behaviour that would take significant custom JS to replicate from scratch.

        

        

          
Modal

          
An accessible overlay dialog with backdrop, focus trapping, and keyboard dismissal — all handled by Bootstrap's JS plugin without any additional code.

        

        

          
Utility Classes

          
Single-purpose classes like .mt-4, .d-flex, and .text-center handle common styling needs inline without writing CSS.

        

        

          
Alerts & Badges

          
Contextual messaging components with semantic colour coding (danger, success, warning, info) that communicate status clearly to all users.

      Critical Evaluation
      
When to Use Bootstrap — and When Not To

      

        A distinction-level understanding of Bootstrap means knowing both its strengths and its limitations.
      


      

        

          ✅
          
Rapid Prototyping
For wireframes, internal tools, or MVPs where development speed matters more than visual uniqueness, Bootstrap is hard to beat.

        

        

          ✅
          
Team Consistency
A shared framework means every developer produces layouts that work the same way, reducing integration bugs.

        

        

          ✅
          
Accessibility Out of the Box
Bootstrap components include ARIA attributes and keyboard interaction patterns — a baseline that's easy to overlook when writing custom components.

        

        

          ⚠️
          
Visual Sameness
Unstyled Bootstrap sites are immediately recognisable. Consumer-facing products that need a distinctive brand often need significant customisation or a different approach.

        

        

          ⚠️
          
Bundle Size
Full Bootstrap imports CSS and JS you may not use. For performance-critical projects, a custom or utility-first CSS approach (like Tailwind) can be leaner.

        

        

          ⚠️
          
Hidden Complexity
Overriding Bootstrap's specificity can be frustrating without understanding CSS cascade. The framework can obscure what's actually happening at the CSS level.
